MERN Stack 技術棧指南

MongoDB + Express.js + React.js + Node.js

什麼是 MERN Stack？

MERN Stack 是一個由四個主要技術組成的全端 JavaScript 技術棧：

MongoDB

NoSQL 資料庫，使用 JSON 格式儲存資料，具有高度擴展性和靈活性。

  • 文件導向資料庫
  • 支援複雜查詢
  • 水平擴展能力強

Express.js

Node.js 的 Web 應用框架，用於建構 API 和處理 HTTP 請求。

  • 輕量級框架
  • 強大的路由系統
  • 豐富的中間件

React.js

用於建構使用者介面的 JavaScript 函式庫，由 Facebook 開發。

  • 組件式開發
  • 虛擬 DOM
  • 單向資料流

Node.js

基於 Chrome V8 引擎的 JavaScript 執行環境。

  • 非阻塞 I/O
  • 事件驅動
  • 豐富的套件生態系統

架構概觀

MERN Stack 採用三層架構：

前端層

React.js 負責使用者介面

應用層

Express.js + Node.js 處理業務邏輯

資料層

MongoDB 負責資料儲存

基本工作流程

Client (React) → API Requests → Server (Express/Node) → Database (MongoDB)
↑                                                                      ↓
JSON Response ← Server Processing ← Database Operations ← Database Query

開發環境設置

必要工具

  • Node.js 和 npm（Node Package Manager）
  • MongoDB Community Server
  • 代碼編輯器（如 VS Code）

基本專案結構

mern-project/
├── client/                 # React 前端專案
│   ├── public/
│   ├── src/
│   └── package.json
├── server/                 # Node.js/Express 後端專案
│   ├── models/            # MongoDB 模型
│   ├── routes/            # API 路由
│   ├── controllers/       # 業務邏輯
│   └── server.js
└── package.json

常用指令

建立新專案

# 建立 React 專案
npx create-react-app client

# 建立 Express 專案
mkdir server
cd server
npm init -y
npm install express mongoose cors dotenv

啟動開發環境

# 前端開發伺服器
cd client
npm start

# 後端伺服器
cd server
npm run dev

最佳實踐

前端

  • 使用 React Hooks
  • 實作狀態管理（Redux/Context）
  • 組件重用

後端

  • RESTful API 設計
  • 適當的錯誤處理
  • 資料驗證

資料庫

  • 適當的索引設計
  • 資料模型正規化
  • 查詢優化

安全性

  • 使用 JWT 認證
  • 資料加密
  • 輸入驗證

常見問題與解決方案

跨域問題

使用 cors 中間件處理

狀態管理

考慮使用 Redux 或 Context API

效能優化

實作快取和代碼分割

部署

使用 Docker 容器化應用
